数据链路层

数据链路和帧

  1. 链路:一个节点到另一个相邻节点的物理线路（有线或无线），而中间没有其他交换节点。

  2. 数据链路:增加了实现通信协议的硬件或软件——网络适配器，包括了数据链路层和物理层的功能。

  3. 协议数据单元：帧。数据链路层把从网络层交下来的数据构成帧，及把接收到的帧中的数据中的数据取出交给网络层。

三个基本问题

  1. 封装成帧：给一段数据添加首部和尾部，有帧定界的作用。注意，数据部分有长度上限——MTU。
  2. 透明传输：不管数据是怎样的比特组合，都能在数据链路层传输。为了避免误识别数据为控制信息，采用字符填充或字节填充。
  3. 差错检测：如比特差错，采取循环冗余检测技术CRC（P76）

点对点协议

  1. 可改变数据内容

  2. 组成：将IP数据报封装到串行链路的方法，建立、配置和测试数据连接的链路控制协议LCP，一套网络控制协议NCP。

  3. 字节填充：将数据中的字节改为7D

  4. 零比特填充：传输过程中加0。
