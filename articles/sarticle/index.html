Basic Routing

accepting a URI and a closure ,and defining routes and behavior

use Illuminate\Support\Facades\Route;

Route::get('/greeting', function () {
    return 'Hello World';
});

The Default Route Files

you will begin by defining routes in your routes/web.php file. The routes defined in routes/web.php may be accessed by entering the defined route’s URL in your browser

use App\Http\Controllers\UserController;

Route::get('/user', [UserController::class, 'index']);

Available Router Methods

The router allows you to register routes that respond to any HTTP verb:

Route::get($uri, $callback);
Route::post($uri, $callback);
Route::put($uri, $callback);
Route::patch($uri, $callback);
Route::delete($uri, $callback);
Route::options($uri, $callback);

register a route that responds to multiple HTTP verbs, register a route that responds to all HTTP verbs

Route::match(['get', 'post'], '/', function () {
    // ...
});

Route::any('/', function () {
    // ...
});

Dependency Injection

use Illuminate\Http\Request;

Route::get('/users', function (Request $request) {
    // ...
});

CSRF Protection

Remember, any HTML forms pointing to POST, PUT, PATCH, or DELETE routes that are defined in the web routes file should include a CSRF token field. Otherwise, the request

<form method="POST" action="/profile">
    @csrf
    ...
</form>

Redirect Routes

Route::redirect('/here', '/there');

customize the status code using the optional third parameter

Route::redirect('/here', '/there', 301);

Route Parameters

Required Parameters

Route::get('/user/{id}', function (string $id) {
    return 'User '.$id;
});

Parameters & Dependency Injection

use Illuminate\Http\Request;

Route::get('/user/{id}', function (Request $request, string $id) {
    return 'User '.$id;
});

Optional Parameters

not always be present in the url

Route::get('/user/{name?}', function (string $name = null) {
    return $name;
});

Regular Expression Constraints

Route::get('/user/{name}', function (string $name) {
    // ...
})->where('name', '[A-Za-z]+');

helper methods allow you to quickly add pattern constraints to your routes

Route::get('/user/{id}/{name}', function (string $id, string $name) {
    // ...
})->whereNumber('id')->whereAlpha('name');

Global Constraints

/**
 * Define your route model bindings, pattern filters, etc.
 */
public function boot(): void
{
    Route::pattern('id', '[0-9]+');
}

Named Routes

Generating of URLs or redirects

Route::get('/user/profile', function () {
    // ...
})->name('profile');
